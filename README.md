# 복지 ONE-GOV — 세대정보 기반 복지 자동판정 통합 플랫폼 (MVP)

흩어져 있는 중앙부처·지자체 복지를 한곳에서 검색·신청하고, 세대정보 기반으로
받을 수 있는 복지를 자동 판정하는 **정부 통합서비스 형태의 시연용 프로토타입**입니다.
일반 국민의 복지 검색·신청부터 주민센터 대리신청, 복지 담당자 처리, 기관 총괄의
세대정보 관리 및 **월 1회 배치 자동판정**, 대시보드 확인까지 전체 흐름을 다룹니다.

- 백엔드/실제 DB 없음 — `localStorage` + JSON 으로 동작
- **복지서비스 데이터**: 복지로 공개정보를 가공한 실데이터 572건(중앙부처 + 서울/경기
  지자체 표본)을 추천·검색·배치에 사용하며, 시연용 대표 사업 7건을 함께 제공합니다.
- 주민·세대 데이터는 모두 **가상 데이터**이며 실제 개인정보가 아닙니다.
- 실제 인증/정부 API/ChatGPT/RAG/지급/알림 발송은 모두 **mock 처리**입니다.

## 디자인

- 황톳빛·단청을 모티프로 한 **따뜻한 색상 팔레트**와 정부 서비스형 식별바를 적용해
  공신력 있는 인상을 줍니다.
- 모든 화면 상단에 **화면 안내 패널**(무슨 화면인지·어떻게 쓰는지)을 제공합니다.
- 고령자·디지털 약자를 위한 접근성(글자 크게/고대비/쉬운 말/다국어)을 지원합니다.

## 실행 방법

```bash
npm install
npm run dev      # 개발 서버 (http://localhost:5173)
npm run build    # 타입체크 + 프로덕션 빌드
npm run preview  # 빌드 결과 미리보기
```

최초 실행 시 `localStorage` 가 비어 있으면 복지사업 7종, 가상주민 100명,
세대 40+개가 자동 생성됩니다. 데이터는 새로고침 후에도 유지됩니다.

## 역할 전환

상단 헤더의 **역할 선택** 드롭다운으로 6개 역할을 전환합니다.
역할에 따라 접근 가능한 메뉴/화면이 달라집니다.

| 역할 | 주요 화면 |
|------|-----------|
| 일반 국민 | 복지 찾기 · 내 신청 현황 · 내 알림 · RAG |
| 주민센터 담당 | 대리 신청 · 대리 신청 이력 · 복지 찾기 |
| 복지 담당 | 신청 관리 · 복지사업 관리 · 세대정보 관리 · SQL |
| 기관 총괄 | 기관 대시보드 · 세대정보 관리 · 배치 처리 · 배치 결과 |
| 복지 총괄 | 전체 대시보드 · 신청 현황 · 배치 결과 |
| 시스템 관리자 | 전체 메뉴 + mock 데이터 초기화 |

## 핵심 시연 흐름

복지 찾기 → 추천받기 → 신청하기 → 담당자 처리 → 세대정보 변경
→ 월 1회 배치 판정 → 자동연결 후보 생성 → 대시보드 반영

- **시나리오 1 (일반 국민)**: 기본 선택된 `가상주민001`(서울·고령·1인·L5)로
  "혼자 사는 70세입니다. 생활비와 돌봄 지원이 필요합니다." 검색 → 생계급여·노인돌봄 추천 → 신청
- **시나리오 5 (배치)**: 기관 총괄 → 월 1회 배치 처리 실행 → L5 세대 자동연결 후보 생성
- **시나리오 6 (SQL)**: "서울에 사는 65세 이상 1인가구 중 소득이 낮은 사람을 찾아줘"
  → mock SQL 생성 + 결과 표시 (`가상주민001` 포함)

## 소득구간 판정 (조건.json)

`src/data/조건.json` 의 1인 세대 기준소득/소득구간을 사용합니다.
2~5인 세대 기준은 프로토타입용 multiplier 로 자동 생성합니다.
**이 multiplier 는 실제 정책 기준이 아니라 mock 기준입니다.**

| 구간 | 의미 |
|------|------|
| L5 | 자동연결 후보 |
| L4 | 수급 가능성 높음 |
| L3 | 조건 확인 필요 |
| L2 | 일부 지원 가능 |
| L1 | 정보 안내 대상 |
| LN | 소득 기준 초과 가능성 |

## 폴더 구조

```
src/
  data/          mockData.ts, 조건.json, welfareServices.json(복지로 실데이터)
  types/         공통 타입
  utils/         storage, conditionRules, residentGenerator, naturalLanguage,
                 matching, mockApi, mockSql, batch, notifications, audit, stats, roles
  store/         AppContext (전역 상태 = localStorage 미러)
  components/
    layout/ accessibility/ citizen/ delegation/ staff/ household/
    batch/ dashboard/ rag/ sql/ admin/
```

비즈니스 로직(조건 판정·추천·자연어·배치·mock SQL)은 모두 `utils/` 로 분리되어
있으며, `localStorage` 접근은 `utils/storage.ts` 로 일원화되어 있습니다.
